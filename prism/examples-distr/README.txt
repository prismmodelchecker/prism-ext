POPTA and property files for the two simple examples in the paper:
-------------------------------------------------------------------------

example1.prism
example1.props

example2.prism
example2.props

PTA versions of these models:

example1_pta.prism
example2_pta.prism

Commands and results (run from the current directory):
 
../bin/prism example1.prism example1.props
Outer bound: 0.5
Inner bound: 0.5

../bin/prism example1_pta.prism example1.props -ptamethod digital
Result: 1.0

../bin/prism example2.prism example2.props
Outer bound: 0.5
Inner bound: 0.5

../bin/prism example2_pta.prism example2.props -ptamethod digital
Result: 0.0

-------------------------------------------------------------------------
PUMP models and property files:
-------------------------------------------------------------------------

pump.prism
pump_deadline.prism
pump.props

NOTE: the model for the deadline version was constructed from the digital clocks MDP model automatically generated by PRISM (further details are included in the model file)

Example commands (run from the pump directory):

../../bin/prism pump.prism pump.props -prop 2 -const N=2,h0=2,h1=2 -gridresolution 2

../../bin/prism pump_deadline.prism pump.props -prop 2 -const N=2,h0=2,h1=2,D=50 -gridresolution 2

The directory also includes an auto file (auto.sh) for generating the logs for the statistics presented in the paper.

-------------------------------------------------------------------------
SCHEDULER models and property files:
-------------------------------------------------------------------------

scheduler.prism (basic model as a POPTA)
scheduler_prob.prism (random delay model as a POPTA)
scheduler.props

The models and property files allow analysis of both reward structures (time and energy)

Example commands (run from the scheduler directory):

../../bin/prism scheduler.prism scheduler.props -prop 2 -const prob1=0.5

../../bin/prism scheduler_prob.prism scheduler.props -prop 2 -const prob1=0.5

The directory also includes an auto file (auto.sh) for generating the logs for the statistics presented in the paper.

-------------------------------------------------------------------------
